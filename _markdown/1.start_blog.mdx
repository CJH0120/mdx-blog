# 블로그를 만든 이유

많고 많은 플랫폼들이 있다 **티스토리**, **네이버**, **velog**, etc..

> 그럼에도 불구하고 나는 왜 개발 블로그를 만들게 되었을까? 한번 알아보자

## 쉽게 할수있는 커스텀 마이징

사람들은 누구나 반복해서 보다 보면 쉽게 물리는 경향이 있다. 티스토리 같은 경우에는 커스텀을 할수있지만 HTML 기반으로 CSS 노가다를 해야하는 불편함이 있다. 네이버와 velog같은 경우는 갇혀 틀에서 벗어날 수없기에 커스텀 마이징을 할수 없게된다.

## 얉은 종속성

몇 년전 **카카오** 서버가 며칠 동안 내려간 적 이 있었다. 지금이야 괜찮지만, 그때는 AI가 발전도 안 했었고, 정보를 찾을 수 있었던건 블로그 하나 였는데 검색을해도, 내가 전에 썼던 글마저 볼 수없는 불편함을 느낄수 있었다. 그래서 언젠가 또 일어날 수 있는 일이라 생각해서 만들기로 했다.

블로그 호스팅은 Vercel에 무료 호스팅을 이용할 예정이다.

> Vercel 서버가 안돌아가면 마찬가지 아닌가요?!

모순이라고 할수있다. 하지만 **local** => **github** => **vercel** 이렇게 호스팅이 되는거라 적어도 내 글은 볼수 있다고 생각해. 종속성이 그나마 낮다고 판단했다.

## 운영비 0원

위에서 말했다 싶이 Vercel에 호스팅을 함으로서 일단 호스팅 비용은 무료이다.

> 그럼 디비서버는 어떻게 구성이 되있을까

디비서버는 없다. 다른 컨텐츠들을 운영해보면서 aws를 이용하여 운영해봤는데, 한달에 6천원 ~ 만원 정도 나온다. 생각을 해보면 블로그 글을 많이 안바꾸는 상태에서 SSR을 쓸 이유도 없고, 로그인이 필요한것도 아니라서 디비서버는 운영 안하기로하고, 로컬에서 **MDX**를 사용하여 파일을 읽어오는 형식으로 구상했다.

그래서 도합 0원의 유지비로 운영을 해 볼 생각이다

## 학습하는 습관 길들이기

학습 습관이 조금은 단순하게 잡혀있었다. **학습 => 적용** 이게 나의 공부하는 방법이였다.

하지만 이제는 **학습 => 적용 => 이해 => 응용** 이 루틴을 적용하기 위해 이해한 내용을 블로그로 작성해보려 한다.

## 문서 쓰는 법 길들이기

회사에 다니면서 느끼는 점은 코딩을 잘짜는 것 보다 , 문서를 잘쓰는 게 더 중요한 것 같다. 예전에는 와 닿지 않았는데, 프로젝트를 만들고, 퇴사도 해보고 하니깐 문서 작성 요령, 습관이 진짜 중요하다는 걸 많이 깨달았다.
그래서 글쓰는 공부와 개발공부의 교집합이 블로그라고 생각이 들어 블로그를 만들고 열심히 작성할 예정이다
