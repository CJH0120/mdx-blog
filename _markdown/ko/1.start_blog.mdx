# 블로그를 만든 이유

현재 많은 플랫폼들이 존재합니다. **티스토리**, **네이버**, **velog**, etc..

> 그럼에도 불구하고 저는 왜 개발 블로그를 만들었을까요? 그 이유를 한번 살펴보겠습니다.

## 커스터마이징의 자유

사람들은 반복해서 보게 되면 쉽게 지루해하는 경향이 있습니다. 티스토리 같은 경우에는 커스터마이징이 가능하지만, HTML과 CSS를 직접 다루는 불편함이 있습니다. 네이버와 velog는 이미 정해진 틀 안에서만 움직일 수 있기 때문에 커스터마이징의 제약이 큽니다.

## 낮은 종속성

몇 년전 **카카오** 서버가 며칠 동안 중단된 적이 있었습니다. 그 당시에는 AI도 발달하지 않았고, 블로그가 유일한 정보원이었는데, 검색해도 이전에 쓴 글을 볼 수 없다는 불편함을 겪었습니다. 이러한 상황이 반복될 수 있다고 생각해, 스스로 운영할 수 있는 블로그를 만들기로 했습니다.

블로그 호스팅은 Vercel을 통해 무료로 진행할 예정입니다.

> Vercel 서버가 중단되면 어떻게 될까요?

일견 모순처럼 보일 수도 있지만, **local** => **github** => **vercel** 의 흐름으로 호스팅이 이루어지기 때문에, 적어도 내 글은 볼 수 있을 것이라 생각합니다. 따라서 종속성이 낮다고 판단했습니다.

## 운영비 0원

앞서 말씀드린 것처럼 Vercel에 호스팅을 하면서 호스팅 비용은 무료입니다.

> 그럼 데이터베이스 서버는 어떻게 구성되어 있나요?

데이터베이스 서버는 운영하지 않습니다. 다른 콘텐츠들을 운영하면서 AWS를 사용해본 경험이 있는데, 월 6천원에서 만원 정도의 비용이 발생하더군요. 블로그 글을 자주 수정하지 않고 SSR을 사용할 필요도 없으며, 로그인 기능이 필요하지 않기 때문에 데이터베이스 서버를 운영하지 않기로 했습니다. 대신, 로컬에서 MDX를 사용하여 파일을 읽어오는 형태로 구상했습니다.

따라서 운영비는 0원으로 유지할 계획입니다.

## 학습하는 습관 길들이기

기존의 학습 습관은 **학습 => 적용**의 단순한 루틴이었습니다.

하지만 이제는 **학습 => 적용 => 이해 => 응용** 의 루틴을 적용하려고 합니다. 이를 위해 이해한 내용을 블로그로 작성해보려 합니다.

## 문서 작성의 중요성

회사에서 느끼는 점은, 코딩 실력보다 문서 작성이 더욱 중요하다는 것입니다. 예전에는 와닿지 않았지만, 프로젝트를 만들고 퇴사도 해보면서 문서 작성 요령과 습관의 중요성을 깊이 깨달았습니다. 그래서 글쓰기와 개발 공부의 교집합으로 블로그를 생각하게 되었고, 열심히 작성해보려 합니다.

### 마치면서

이렇게 다섯 가지 이유로 개인 블로그를 만들게 된 배경을 간략히 설명드렸습니다. 다음 편에서는 이 블로그가 어떻게 구성되었는지, 어떤 라이브러리를 사용했는지에 대해 다뤄보겠습니다.
