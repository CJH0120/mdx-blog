## 개발 블로그를 만든 이유

1. **기록하는 습관 기르기**

   - 코딩을 시작한 이후, 학습 내용을 기록한 적이 거의 없었습니다. 이번 기회에 기록하는 습관을 기르며, 개발 과정에서의 경험과 배운 내용을 체계적으로 정리하고 기록하고자 합니다.

2. **문서 작성 능력 향상**

   - 잘 정리되고 논리적으로 구성된 문서를 작성하는 연습을 통해 문서 작성 능력을 개선해 보려 합니다.

3. **생각 정리 및 정보 공유**
   - 학습한 내용을 정리하여 다른 사람들도 쉽게 이해할 수 있도록 블로그에 공유하고자 합니다.

<br />

## 벨로그나 티스토리를 선택하지 않은 이유

1. **티스토리**: 커스터마이징이 어렵다.
2. **벨로그**: 서버 상태가 불안정하다.

> 그렇다면 내 블로그는 어떨까?

데이터베이스를 사용하지 않고, 스타일링은 자유롭게 커스터마이징할 수 있도록 설정했습니다. Vercel의 무료 서버를 이용해 앞으로 0원의 비용으로 블로그를 운영해 볼 예정입니다.

<br />

## 블로그 기술 스택

이 블로그는 Next.js를 이용하여 로컬에서 MDX 파일을 읽어오는 방식으로 제작했으며, 스타일링에는 SCSS를 사용했습니다.

### Next.js를 사용한 이유

1. **SEO 최적화**
2. **빠른 렌더링**
3. **익숙한 프레임워크**

이렇게 세 가지 이유로 Next.js를 선택하게 되었습니다.
