# 개발 블로그를 만든 이유

1. 기록하는 습관 길들이기

   - 코딩을 시작한 이후로, 저의 학습 내용을 기록한 흔적이 전혀 없었습니다. 그래서 이번 기회에 기록하는 습관을 길들이고자 합니다. 블로그를 통해 개발 과정에서의 경험이나 학습한 내용을 체계적으로 정리하고 기록하려고 합니다.

2. 문서 작성 능력 향상

   - 정보가 잘 정리되고 논리적으로 구성된 문서를 작성하는 연습을 통해 문서 작성 능력을 개선해보려 합니다.

3. 생각 정리 및 정보 공유
   - 내가 학습한 내용을 정확히 정리하여, 다른 사람들이 쉽게 이해할 수 있도록 블로그에 공유하고자 합니다.

이렇게 정리할 수 있을 것같다.

### 벨로그나 티스토리를 선택 안한 이유

1. 티스토리는 커스텀 하기 힘들다.
2. 벨로그는 서버 상태가 불안정하다.

> 그렇다면 내 블로그는 어떠한가?

디비는 사용하지 않고, 스타일링은 쉽게 커스텀할 수 있도록 내 마음대로 설정했습니다. Vercel의 무료 서버를 이용하여, 앞으로 0원으로 블로그를 운영해 보려고 합니다.
